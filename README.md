# Production Support Metrics Dashboard (Power BI)

## 📌 Objective  
This dashboard provides a comprehensive view of production support operations by analyzing incidents, requests, change tasks, and technical activities.  
It helps monitor SLA performance, track workload distribution, and support operational decision-making.

---

## 🛠 Tools Used  
- Power BI  
- Microsoft Excel  

---

## 📂 Dataset  
A dummy ITSM ticket dataset was created using Excel for portfolio purposes.  
The dataset contains the following fields:
- Ticket ID  
- Date  
- Type (Incident, Request, Change Task, Technical Activity)  
- Priority  
- Status  
- Category  
- Resolved By (Resource)  
- SLA TTO (Time to Own)  
- SLA TTR (Time to Resolve)  
- Shift (Day / Night)  

⚠️ No real company data is used in this project.

---

## 📈 Key Metrics (KPIs)  
- Total Incidents  
- Total Requests  
- Total Technical Activities  
- Total Change Tasks  
- SLA Compliance %  
- TTO Compliance %  
- TTR Compliance %  
- Total Tasks  
- Day Shift vs Night Shift distribution  

---

## 🖥 Dashboard Features  

### 1. Filters  
The dashboard supports dynamic filtering by:
- Resolved By  
- Category  
- Status  
- Type  
- Priority  
- Date range  

---

### 2. Trend Analysis  
- Daily ticket trend to identify workload patterns  
- Helps understand peak operational days  

---

### 3. Resource Performance  
- Tickets handled by each resource  
- Enables workload comparison and capacity planning  

---

### 4. Ticket Breakdown  
- Tickets by Priority  
- Tickets by Category  
- Tickets by Status  
- Matrix view for status vs priority  

---

### 5. Operational View  
- Detailed ticket table with:
  - Date  
  - Ticket ID  
  - Resolved By  
  - Status  
  - Description  

---

### 6. Shift & On-Call View  
- Shift schedule calendar (Day & Night shifts)  
- On-call contact and escalation details  
- Supports operational continuity planning
